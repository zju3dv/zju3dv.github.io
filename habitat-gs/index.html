TL; DR

We present Habitat-GS, a navigation-centric embodied AI simulator based on 3DGS and dynamic gaussian avatars. Compared to traditional mesh-based simulators, our simulation environment delivers photorealistic rendering. Habitat-GS is extended from facebook research's Habitat-Sim, and we integrated 3DGS scene rendering backend and a drivable gaussian avatar module into it. Our simulation platform is fully open-source and maintains complete compatibility with Habitat ecosystem. By introducing gaussian splatting to embodied simulators, we hope our work can facilitate future embodied AI research.

Navigation Episodes

Habitat-GS is fully compatible with the Habitat ecosystem, including Habitat-Lab tasks, training, and evaluation APIs.
This means users can directly train and evaluate navigation agents in GS scenes on Habitat-Lab. Below we show five navigation tasks ran on Habitat-Lab using Habitat-GS as its simulator.

Point Goal Navigation: Given RGB-D observations and point-goal signal, navigate to the goal location.

Image Goal Navigation: Given an image, navigate to the location where the image is taken.

Object Goal Navigation: Given the semantic ID of an object, navigate to the corresponding location.
In video, target objects are sofa and plant respectively.

Avatar-aware Point Goal Navigation: Navigate to the destination without colliding with human avatars.

Human Tracking Navigation: Agent is supposed to track human avatar continuously.

Method

System overview of Habitat-GS. From left to right: Asset Preparation, where 3DGS scene assets and gaussian avatar assets are prepared respectively. Habitat-GS Simulation Environment, where the render engine performs 3DGS rasterization for scene gaussians and LBS deformation followed by rasterization for avatar gaussians, producing RGB-D observations. The NavMesh blocking module retrieves pre-computed proxy capsules at runtime and injects them into the NavMesh for obstacle blocking. Embodied Agent Loop, where the agent policy consumes sensor observations, executes actions, and receives rewards shaped by both navigation success and avatar collision.

Architecture

System architecture of Habitat-GS. Habitat-GS is built as a non-intrusive extension to Habitat-Sim. The system adopts a visual-navigation decoupling design principle, separating the visual rendering modules handled by the CUDA-based 3DGS rasterizer and LBS deformation, from the navigation module managed by the traditional NavMesh and injected proxy capsules. This allows for photorealistic agent observations without modifying the core Habitat-Sim navigation logic. For more details, please refer to our paper.
